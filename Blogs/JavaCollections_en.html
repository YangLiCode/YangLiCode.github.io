Java Collections

You may have used ArrayList to solve coding problems. In fact, Java has provided many collection classes that are essential for developers. Let’s take a look!

That’s a lot of collection classes! But don’t worry. We’ll look at each one of them. And by the end of the tutorial, you’ll feel more familiar with them.

Collections and Map

There are 2 fundamental interfaces in Java Collections: Collections and Map. Collections are for single value lists/sets. Maps are for key-value pairs.

We often write code like this:

List myList = new ArrayList<>();

The code above created a new ArrayList Object and assigned that to a List variable. This is because if we change our mind, we can assign a different implementation of List later.

Because Collection is the interface for all its subclasses, we could use call it’s methods from all of its subclasses. Here are some common ones:

methods Descriptions
public boolean add(E e) Adds an element to this collection
public void clear() Removes all elements from this collection
public boolean remove(E e) Removes an element from this collection
public boolean contains(Object obj) Returns true if this collection contains the specified element
public boolean isEmpty() Returns true if this collection contains no elements
public int size() Returns the number of elements in this collection
public Object[] toArray() Returns an array containing all of the elements in this collection
